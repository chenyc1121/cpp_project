# 🏫 C++ OOP 大富翁 (Monopoly)

北大校园主题的 C++ 面向对象编程概念大富翁游戏，基于 Qt6/C++17 开发。

## 演示视频链接

## 🎮 快速开始

```bash
cmake -B build/Desktop-Debug -G "Unix Makefiles" -DCMAKE_BUILD_TYPE=Debug
cmake --build build/Desktop-Debug
./build/Desktop-Debug/OOP_project
```

## 🎯 游戏概述

2-4 名玩家在北大校园地图（28格）上掷骰前进，购买地产、收取租金，最后一名未破产的玩家获胜。

- **初始资金**：¥20,000
- **经过起点**：¥5,000（恰好停在起点 ¥10,000）
- **胜利条件**：使其他所有玩家破产

## 🏷️ 地块类型

### 起点 (START)
- 索引 0，经过获得 ¥5,000，恰好停在起点获得 ¥10,000

### 地产 (PROPERTY) — 可购买、建房、收租

6 个颜色组，每组集齐后可建房（棕色/深蓝组 2 块，其余 3 块）：

| 颜色组 | 地块 | 价格 | 建房费 |
|--------|------|------|--------|
| 棕色 (BROWN) | 45甲 | ¥1,200 | ¥800/栋 |
| 浅蓝 (LIGHT_BLUE) | 35楼、智华楼 | ¥1,600-1,800 | ¥1,000/栋 |
| 橙色 (ORANGE) | 博雅塔 | ¥2,600 | ¥1,400/栋 |
| 红色 (RED) | 未名湖、图书馆 | ¥2,800-3,000 | ¥1,600/栋 |

**租金表**（每块地产有 6 级租金：空地 / 1栋 / 2栋 / 3栋 / 4栋 / 旅馆）
- 点击格子中间可查看完整的 6 级基础租金表

**建房规则**：拥有同色组所有地块后，可在自己回合踩中自己地产时建房。1-4 栋逐级升级，第 5 次升级为旅馆。

### 静态成员变量格 (STATICVAL)
- 黄色组（3块）：静态成员变量·A / B / C
- 定价 ¥2,000-2,400，建房费 ¥1,000/栋
- **核心机制**： 通过点击 `[i]` 按钮可查看 StaticvariableTile 类定义伪代码查看

### 虚函数格 (VIRTUALFUNC) — 5块，模拟 C++ 虚函数多态

**虚函数卡互动机制**：踩中别人虚函数格时，由**地主**（非踩中方）决定是否使用虚函数卡：
- **无卡** → 收取基类租金（标准）
- **有卡** → 地主可选择基类（安全）或派生类（高收益）租金

虚函数卡需要结合代码判断能否使用，若无法使用则卡片会被判定为无效

### 迭代器格 (ITERATOR) — 4块，模拟 C++ 迭代器

三教 / 二教 / 理教 / 一教（价格 ¥1,500-2,000）

- **租金**：`250 × 2^(n-1)`（拥有 1→¥250, 2→¥500, 3→¥1,000, 4→¥2,000）
- **迭代器卡传送**：在 4 个迭代器格之间按向量顺序移动，点击 `[i]` 按钮可查看 vector 迭代器伪代码


### 其他功能格

| 类型 | 效果 |
|------|------|
| **问答格 (QA)** ×4 | 回答 C++ OOP 选择题，答对 65% 概率获得效果卡 |
| **商店 (SHOP)** ×1 | 购买效果卡（角格 7，"麦叔的铺子"） |
| **29楼地下室 (SHOP_ENTRANCE)** ×1 | 可选择传送到商店（角格 21） |
| **上机课 (COMPUTER_LAB)** ×1 | 须回答 C++ 题目，答对必得效果卡，下回合跳过（角格 14） |
| **税收 (TAX)** ×2 | 农园 ¥2,000 / 燕南 ¥3,000 |

## 🃏 效果卡系统

5 种消耗品卡牌，替代传统机会/命运卡：

| 卡牌 | 商店价 | 效果 | 获取途径 |
|------|--------|------|---------|
| **再丢一次 (ROLL_AGAIN)** | ¥500 | 掷骰后可再掷一次 | 商店/QA/上机课 |
| **万能骰子 (UNIVERSAL_DICE)** | ¥1,000 | 自选两个骰子点数（1-6） | 商店/QA/上机课 |
| **虚函数卡 (VIRTUAL_FUNCTION)** | ¥800 | 在虚函数格上可切换基类/派生类行为 | 商店/QA/上机课 |
| **跳过卡 (SKIP_EFFECT)** | ¥1,500 | 跳过当前格的负面效果（税收、他人收租） | 商店/QA/上机课 |
| **迭代器卡 (ITERATOR_CARD)** | ¥600 | 在迭代器格间传送（3种子类型） | 商店/QA/上机课 |

**迭代器卡子类型**（随机分配）：若迭代器卡的类型与所选操作不匹配则卡片会被判为无效

## 📚 问答系统

题库包含 **50 道** C++ 面向对象选择题，涵盖：
- 运算符重载、类与对象、构造/析构、继承与访问控制
- 虚函数与多态、纯虚函数/抽象类
- 函数模板、类模板、STL 容器与算法
- C++11 新特性（auto、lambda、智能指针、移动语义）
- 指针与引用、函数重载决议

答对 QA 格题目有 65% 概率获得随机效果卡（5种均匀分布）；上机课答对 100% 获得。

## 🎲 游戏流程

### 状态机

```
开始新游戏 → PRE_ROLL（等待掷骰）→ ROLLING（掷骰中）→ MOVING（移动中）→ LANDING（落地处理）→ TURN_END（回合结束）→ PRE_ROLL...
```

### 每回合流程

1. **PRE_ROLL**：检查是否跳过回合（上机课效果）→ 检查万能骰子卡 → 掷骰
2. **ROLLING**：显示骰子结果 → 检查再掷一次卡 → 进入移动
3. **MOVING**：玩家沿棋盘移动，经过起点自动领取 ¥5,000
4. **LANDING**：触发地块效果（购买/收租/问答/商店等）
5. **TURN_END**：检查对子（连续对子获得额外回合）→ 下一位玩家

### 关键规则

- **掷出对子**：获得额外一个回合（无次数限制，无监狱机制）
- **购买/建房**：需要玩家决策，所有弹窗为模态对话框
- **资金不足**：无法购买/建房时自动跳过
- **等待决策**：landOn 触发弹窗后，`m_waitingForDecision` 标志防止自动结束回合

## 💰 破产系统

- 付不起租金时，将剩余所有现金转给债主，余额清零
- 自动触发破产清算（`liquidate`）：
  - 所有地产归还银行（清除房屋/旅馆）
  - 所有迭代器格清空所有权
  - 所有效果卡丢弃
  - 标记为破产状态，跳过后续回合
- 最后一名未破产玩家获胜

## 🖥️ 界面操作

### 棋盘交互
- **点击格子中间**：显示 tooltip（完整基础租金表、价格、建房费、所有者）
- **点击 `[i]` 按钮**（标题栏）：显示教学说明（伪代码/类型定义）
- **点击 `[...]` 按钮**（右下角）：显示地块详情说明

### 菜单
- **新游戏 (Ctrl+N)**：开始新游戏（2-4 人）
- **DEBUG 模式**：开启后每次掷骰可手动输入点数
- **规则说明**：游戏内查看规则摘要
